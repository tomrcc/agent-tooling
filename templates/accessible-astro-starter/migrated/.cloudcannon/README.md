# Welcome to Accessible Astro Starter

This is your site's content management dashboard. Here you can edit pages, blog posts, and portfolio projects.

## Quick Links

- [Homepage](cloudcannon:collections/pages)
- [Blog Posts](cloudcannon:collections/blog)
- [Portfolio Projects](cloudcannon:collections/projects)

## Editing Guide

### Pages

The [Pages](cloudcannon:collections/pages) collection contains your site's main pages. The homepage has editable sections including the hero banner, features grid, FAQ, community members, and statistics counters. You can add, remove, and reorder items in each section.

To edit the homepage, click on it in the Pages collection. Use the visual editor to make inline text and image changes, or switch to the data editor to manage structured sections like features and FAQ items.

### Blog

The [Blog](cloudcannon:collections/blog) collection contains your blog posts. Each post has a title, description, author, date, featured image, and tags. Click "+ Add" to create a new post — you'll start in the content editor where you can write your article.

### Portfolio Projects

The [Projects](cloudcannon:collections/projects) collection showcases portfolio items. Each project has a title, author, description, and tags. Project content is written in MDX and supports special components:

- **Breakout Image** — a full-width image that extends beyond the content area
- **Block Quote** — a styled quotation with optional author attribution

Insert these from the snippet toolbar button (the puzzle piece icon) when editing project content.

### Site Pages

The [Site Pages](cloudcannon:collections/site_pages) section contains standalone pages like the markdown demo page and accessibility statement. These are edited through the visual editor.

## Tips

- Use the **visual editor** for quick text and image changes directly on the page
- Use the **content editor** for writing blog posts and long-form content
- Use the **data editor** to manage structured data like feature lists, FAQ items, and team members
- Changes are saved to your repository and trigger a new build automatically
