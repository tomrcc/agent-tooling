# 🔮 Questions to ponder 🔮

## Content Blocks 

Whats the ideal structure for pages in a pages directory? Do we still want to go with the array of components approach, or are we ok with making components static in the layout and connecting the props to frontmatter? 

It seems like its easier to make hardcoded components in layouts editable now with editable regions than it was with Bookshop, and I'm wondering if its time for a conversation about what approach we would choose if we didn't know anything about a site, but had to choose.

It does make page building more complicated not just having every page be an array of components. We *could* lean more heavily on add options and schemas for page building if we wanted to encourage that path.
