# Audit

Guidance for analyzing a site before migrating it to CloudCannon.

This doc will be filled in as we work through real template migrations. Expected topics:

- How to detect the SSG and version
- Mapping content directories to potential CloudCannon collections
- Identifying components, layouts, and partials
- Flagging patterns that need special handling (dynamic routes, API-driven content, etc.)
- Output format for audit findings that later phases can consume
